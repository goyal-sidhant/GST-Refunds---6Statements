================================================================================
2. PURPOSE & LEGAL BASIS
================================================================================

This template is for zero-rated exports of goods/services made WITHOUT payment
of tax, where refund of accumulated ITC is claimed under Section 54(3) read
with Rule 89(4).

Since no output tax was paid, there are NO tax amount columns (no IGST/CGST/SGST
block). The template captures:
  - Export invoice details
  - For Goods: customs evidence (Port Code, Shipping Bill, EGM)
  - For Services: banking evidence (BRC/FIRC)

REFUND CATEGORY: Accumulated ITC.
The Goods/Services flag (Column F) drives which evidence fields are relevant.

FILE NAME HINT: "MultipleBRC" in the file name indicates this version supports
multiple BRC/FIRC entries linked to a single invoice.

================================================================================
5. GOODS vs SERVICES LOCK/UNLOCK (VBA BEHAVIOUR)
================================================================================

WHEN "G" IS SELECTED:
  - Columns G–L (Port Code through EGM Date) remain editable.
  - Columns M–O (BRC/FIRC) remain editable (optional for goods).
  - VBA PREVENTS switching from S to G if Port Code (Col G) already has data.
    Shows MsgBox: "Please clear the Details before selecting Goods"
    and reverts selection back to "S".

WHEN "S" IS SELECTED:
  - Columns G–L are LOCKED and display "DISABLED" in grey
    (ColorIndex = 15, Font.ColorIndex = 15 to hide text).
  - Columns M–O (BRC/FIRC) are UNLOCKED for banking evidence.

WHEN "CLEAR" IS SELECTED:
  - All columns F through L are cleared and unlocked.
  - Colors reset to white (ColorIndex = 2).

================================================================================
6. KEY VALIDATION RULES
================================================================================

AMOUNTS:
  - Document Value: max 15 + 2 decimal digits, >= 0.
  - FOB Value: SPECIAL — allows 15 + 4 decimal digits (FOB_VALUE_REGEX).
    This is stricter than other statements which use 15+2.
  - BRC/FIRC Value: 15 + 2 decimal digits.

SHIPPING / CUSTOMS:
  - Port Code: exactly 6 alphanumeric characters.
  - Shipping Bill Number: 3-7 digits (SHP_REGEX = ^[0-9]{3,7}$).
  - EGM Reference: 1-20 alphanumeric (EGM_NO_REGEX = ^[a-zA-Z0-9]{1,20}$).

DATES:
  - All dates validated for calendar correctness including leap years.
  - >= 01-Jul-2017 and <= current date.
  - Shipping bill date and EGM date also validated independently.

BRC/FIRC:
  - BRC No.: 2-30 alphanumeric characters.
  - BRC Date: standard dd-mm-yyyy validation.
  - BRC Value: numeric, >= 0.

DUPLICATE DETECTION:
  Key = DocType suffix (last 4 chars) + "_" + DocNo + "_" + FY
  If invoice month is Jan-Mar, FY year is decremented by 1.
  Dictionaries: DUPLICATE_REC and DUPLICATE_REC1

FOB_VALUE_REGEX   | ^([0-9]{0,15})([.][0-9]{0,4})?$              | FOB value (4 decimals!)

BRC_VAL_REGEX     | ^([0-9]{0,15})([.][0-9]{0,2})?$              | BRC/FIRC value

PER-ROW JSON KEYS:

JSON Key    | Source Col | Applies To
------------|-----------|------------------------------------------
sno         | A         | Both G and S
docType     | B         | Both
inum        | C         | Both
idt         | D         | Both
val         | E         | Both
type        | F         | Both ("G" or "S")
sbpcode     | G         | Goods only
sbnum       | H         | Goods only
sbdt        | I         | Goods only
fobValue    | J         | Goods only
egmref      | K         | Goods only
egmrefdt    | L         | Goods only
brcfircnum  | M         | Services (and optionally goods with BRC)
brcfircdt   | N         | Services
brcfircval  | O         | Services

JSON GENERATION — CONDITIONAL PATHS:
  - BRC-only row (Col M has data, Col B is blank):
    → Generates sno + brcfircnum + brcfircdt + brcfircval only.
  - Document-only row (Col B has data, Col M is blank):
    → For S: sno, docType, inum, idt, val, type
    → For G: same + sbpcode, sbnum, sbdt, fobValue, egmref, egmrefdt
  - Full row (both Col B and Col M have data):
    → Two JSON chunks written — invoice portion + BRC portion.

APPENDIX A: GST PORTAL TUTORIAL LINK
================================================================================

Official Tutorial URL:
https://tutorial.gst.gov.in/userguide/refund/Refund_of_ITC_paid_on_Exports_of_Goods_and_Services.htm

Portal Workflow (summarised from tutorial.gst.gov.in):
  1. Login to GST Portal → Services → Refunds → Application for Refund
  2. Select the appropriate refund type from the dropdown
  3. Select the Tax Period and click CREATE REFUND APPLICATION
  4. IMPORTANT: All applicable returns (GSTR-1, GSTR-3B) must be filed
     for the period before refund application can be created
  5. Download the Offline Utility (.xlsb file) from the portal
  6. Fill in the data in the Excel template
  7. Click 'Validate & Calculate' to check for errors
  8. Rectify any errors shown in the Error column
  9. Click 'Create File To Upload' to generate the JSON file
  10. Upload the JSON file on the GST Portal
  11. Click 'Validate Statement' on the portal
  12. If invalid documents exist, download error sheet, fix, re-upload
  13. Once validated, proceed to fill refund computation table
  14. Select bank account, upload supporting documents
  15. Save the application (valid for 15 days)
  16. File with DSC or EVC
  17. ARN is generated and sent via email/SMS

KEY PORTAL NOTES:
  - System does NOT validate period of limitation (Section 54(1)).
    Taxpayer must ensure filing within 2 years from relevant date.
  - Same invoice cannot be reused in another refund application.
  - If deficiency memo is issued or application withdrawn, invoices
    become available for reuse.
  - Up to 15 supporting documents (5 MB each) can be uploaded.
  - Nil refund can be filed if no refund is to be claimed for a period.

Function getDataFromSheet()
    Dim currentWS As Worksheet
    Set currentWS = ActiveSheet
    Dim lastCell As Range
    
        
    With Sheets("RFD_STMT03")
    Set lastCell = Range("A13:O10012").Find(What:="*", _
                          After:=.Range("A13"), _
                          Lookat:=xlPart, _
                          LookIn:=xlFormulas, _
                          SearchOrder:=xlByRows, _
                          SearchDirection:=xlPrevious, _
                          MatchCase:=False)
    End With
    
    If lastCell Is Nothing Then
        LASTROW = 12
    Else
        LASTROW = lastCell.Row
    End If

    If lastCell Is Nothing Then
        STARTROW = 0
        ENDROW = 0
        WS_DATA = Null
    Else
        WS_DATA = currentWS.Range("A13:P" & LASTROW).Value
        STARTROW = LBound(WS_DATA, 1)
        ENDROW = UBound(WS_DATA, 1)
    End If
        
    Set currentWS = Nothing
End Function

    
    

Function writeDataToSheet()
    
    Range("A13:P" & LASTROW).Value = WS_DATA
    
    'To Adjust COlumn I (Errors) height such that error messages are completely visible.
    Columns("P:P").AutoFit
    
    'Select the cells which are below the last row containing value in coulmn A. Clear those selcted cells.
    Range(Cells(LASTROW + 1, "P"), Cells(LASTROW + 1, "P").End(xlDown)).ClearContents
    
    
End Function

Sub autoGenSerial()

    Dim serialNo As Integer
    Dim Row As Integer
    
    serialNo = 1
        
    For Row = STARTROW To ENDROW
        'Debug.Print WS_DATA(Row, 2)
        
        If WS_DATA(Row, 2) <> "" Or WS_DATA(Row, 13) <> "" Then
           WS_DATA(Row, 1) = serialNo
           serialNo = serialNo + 1
          End If
          
    Next Row
        
End Sub

Function checkAndCreateJson()
    Dim errorCount, addCount, rowNo As Integer
    Dim errMsg As String
    
    errMsg = " Exists in the sheet. Please click Create File To Upload after resolving the errors"

    addCount = 0
      
    If Not (STARTROW = 0 And ENDROW = 0) Then
       For rowNo = STARTROW To ENDROW Step 1
  
            If Not WS_DATA(rowNo, 16) = "" Then
                errorCount = errorCount + 1
            End If
    
        Next
     End If
      
    If (STARTROW = 0 And ENDROW = 0) Then

        MsgBox "Please type atleast one record to create file to upload", vbCritical
      
    ElseIf errorCount = 0 Then
    
        If Cells(6, "E") <> "" Then
                MsgBox "Valid GSTIN is Mandatory", vbCritical
        ElseIf Cells(7, "E") <> "" Then
                MsgBox "Valid From Return Period is Mandatory", vbCritical
        ElseIf Cells(8, "E") <> "" Then
                MsgBox "Valid To Return Period is Mandatory", vbCritical
        Else
                autoGenSerial
                createJsonFile
        End If
    
    ElseIf errorCount = 1 Then
        MsgBox errorCount & " error" & errMsg, vbCritical
    Else
        MsgBox errorCount & " errors" & errMsg, vbCritical
    End If
    
End Function
